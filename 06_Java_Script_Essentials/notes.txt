--> ** Javascript : It is a programming language used to make web pages interactive and dynamic. It is case-sensitive language. It is clint side language(execute through browser.) , when node js introduced it becames server side also it means we can run js outside the browser. 
   *** Javascript is used for both client and server side programming. 
      -> Client side : execute by the browser. (front-end)
      -> Server side : execute by the node js. node js runtime environment. (back-end) 

    -> Javascript is work on ECMAScript standard (ES) ,ECMAScript is a international standard 
        ** ECMAScript = Javascript 
    
      ->ECMAScript have many versions.
        ** ES1 , ES2 , ES3 , ES4 , ES5 , ES6 , ES7 , ES8 , ES9 , ES10 , ES11 , ES12 , ES13 , ES14 , ES15 , ES16 , ES17 , ES18 , ES19 , ES20 , ES21 , ES22 , ES23 , ES24 
        *** ES6 is the most important version of ECMAScript. (new features are introduced in ES6)
    
    -> when we execute javascript code , it execute in a browser
        ->  Every browser have a javascript engine which execute the javascript code.
            ->  Google Chrome have V8 engine.
            ->  Mozilla Firefox have SpiderMonkey engine.
            ->  Microsoft Edge have Chakra engine.
            ->  Safari have JavaScriptCore engine.

        -> Javascript is case sensitive language 
        -> extension of javascript file is .js
        ->How we run js code in html
            -> <script src="filename.js"></script>  --> external file 
            -> <script> </script>  --> internal file 
          

    -> How we print data in a browser (output)
       1) document.write("Hello World")
       2) console.log("Hello World") --> open inspect element 
       3) alert("Hello World") --> show popup 
       4) document.getElementById("id").innerHTML = "Hello World";
       5) prompt("Enter your name") --> show input box (we can also print the input value)
       6) document.getElementById("id").textContent = "Hello World";

    -> console : here we find all javascript error messages, 
       -> when error occure in a webpage , webpage freeze 
       -> in console we can also write js code and execute it 
       -> console have 5 tabs
         1) console
         2) source
         3) network
         4) performance
         5) application

         console.log("hello") it will display output in console tab 
         alert("hello") it will display output in popup box 
         
         comments : describe the code (hidden in browser)
            -> //  --> single line comment
            -> /* */  --> multi line comment
            -> /** */ --> multiple line comment (documentation)


      -> how Javascript code is exexuted
        -> Code is Loaded
        -> Execution context is created 
        -> Code runs line by line 
        -> Functions go to callstack 
        -> Event loop manage execution flow 


        1. Js engine execute the js code 
           - Read js Code
           - Convert it into machine code (Binary Code) 
           - Execute it 
        2. Execution context
           - when js run , its create an executin context
             -- 3 Typers of execution context
                1) Global Execution Context :
                2) Function Execution Context 
                3) Eval Execution Context 
        3. Javascript is run in 2 phases 
           - phase 1 : Memory Creation Phase(Hoising)
              --Variables and funtions are store in a memory 
           - phase 2 : Code Executin phase 
              --Actual Values are assigned 
              

      -> *** Javascript is a single threaded language (it can execute one task at a time) 
         - Synchronous : code is executed line by line 
         - Asynchronous : code is executed when the task is completed (non-blocking) 

            callstack --> Event Loop --> Callback Queue --> Web API
            callstack <-- Event Loop --> Callback Queue --> Web API

--> Download and install node js in system 
    - node js is a runtime environment for executing javascript code outside the browser. 
    - it is built on google chrome's v8 engine. 
    - it is used for server side programming. 
    - it is open source and cross platform. 
    - it is used for building web applications, command line tools, and desktop applications.
    - when we install node js in our system , it also install npm (node package manager) which is used for managing packages and dependencies in node js projects.
          how to check node js and npm version in terminal
          - node -v
          - npm -v
