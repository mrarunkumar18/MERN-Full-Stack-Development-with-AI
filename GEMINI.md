# Project Instructions: MERN Full Stack Development with AI

This document outlines the architecture, conventions, and workflows for this repository. These mandates take precedence over general defaults.

## 🎯 Project Overview
A structured learning journey from web fundamentals (HTML/CSS) to JavaScript essentials, leading into MERN stack development (MongoDB, Express, React, Node) and AI-powered applications, with a long-term goal of specializing in Cybersecurity.

## 📂 Directory Structure Conventions
- **Numbered Modules (`00_` to `XX_`):** Used for chronological learning phases. Each folder should contain relevant HTML/CSS/JS files and a `notes.txt` or `Notes.txt` file summarizing the theory.
- **`Projects/`:** Contains standalone functional projects (e.g., Portfolio Website).
- **`Resources/`:** Practice-focused exercises and deep dives into specific languages/tools (e.g., `Resources/JavaScript`).

## 🛠 Coding Standards & Style

### General
- **Indentation:** 2 spaces for all file types (HTML, CSS, JS).
- **File Naming:** Use `kebab-case` or `snake_case` for consistency. Avoid spaces in filenames.

### HTML
- Always use semantic HTML5 elements (`<header>`, `<main>`, `<footer>`, `<section>`, `<article>`, `<nav>`).
- Include standard metadata and responsive viewport tags in all `index.html` files.

### CSS
- Prefer external stylesheets (`style.css`) for project files.
- Use internal or inline CSS only for isolated concepts or quick experiments in learning modules.
- Utilize Bootstrap 5 (CDN or Local) for responsive layouts where specified.
- Use CSS Grid and Flexbox for modern, responsive designs.

### JavaScript
- Use modern ES6+ syntax (`const`/`let` instead of `var`, arrow functions, template literals).
- Maintain clear, descriptive variable and function names.
- Document logic with concise comments where necessary.

## 🔄 Workflow
- **Learning New Concepts:** Create a new numbered folder, add theory notes in `note.txt`, and implement practical examples.
- **Practice Exercises:** Use the `Resources/` directory for small-scale practice problems.
- **Building Projects:** Use the `Projects/` directory for end-to-end application development.
- **Commiting:** When requested, provide clear commit messages that explain the "why" of the change.

## 🤖 AI Integration
- This project utilizes Gemini CLI for code generation, refactoring, and architectural guidance.
- When asking for help, specify if the context is a "Learning Module", "Resource Practice", or "Project Task".
